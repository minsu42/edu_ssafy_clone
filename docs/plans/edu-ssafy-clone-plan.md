# edu.ssafy.com Clone Coding Plan

**Goal:** edu.ssafy.com의 핵심 사용자 경험과 디자인을 재현하는 교육 포털 클론을 만든다.

**Reference:** `DESIGN.md`

**Scope:** 전체 서비스를 1:1로 복제하기보다, 주요 페이지의 레이아웃/디자인/인터랙션을 중심으로 구현한다.

---

## 1. 기준 정리

- `DESIGN.md`를 기준 디자인 시스템으로 사용한다.
- 실제 사이트에서 확인한 주요 화면을 우선순위로 삼는다.
  - 메인 대시보드
  - 마이캠퍼스
  - 강의실 / 주차별 커리큘럼
  - 커뮤니티 / 설문조사
- 민감한 실제 데이터는 사용하지 않고 더미 데이터로 대체한다.

## 2. 구현 범위 결정

먼저 클론의 목적을 정한다.

- 디자인 재현 중심인지
- 실제 로그인/상태관리까지 포함할지
- 반응형 모바일까지 포함할지
- 백엔드/API를 만들지, 프론트엔드 목업으로 끝낼지

이 결정에 따라 작업량이 크게 달라진다.

## 3. 화면 구조 설계

공통 레이아웃을 먼저 만든다.

- Header / GNB
- Hero Banner
- Secondary Tabs
- Main Container
- Footer
- Floating e-book Button

이후 페이지별 화면을 붙인다.

- Home Dashboard
- MyCampus
- Classroom Weekly Curriculum
- Community Survey

## 4. 디자인 시스템 적용

`DESIGN.md`의 토큰을 기반으로 공통 스타일을 만든다.

- 색상
- 폰트
- 간격
- 카드
- 버튼
- 탭
- 배지
- 테이블
- 페이지네이션

가능하면 직접 CSS 변수 또는 Tailwind theme로 변환해 사용한다.

## 5. 데이터 모델링

실제 서버 연동 전에는 더미 데이터를 먼저 정의한다.

- 사용자 정보
- 알림 목록
- 장학포인트/EXP
- 커리큘럼 일정
- Quest/평가 목록
- 학습자료 목록
- 설문 목록

데이터 구조를 먼저 맞춰두면 이후 백엔드/API 연동이 쉬워진다.

## 6. 페이지 구현 순서

1. 공통 레이아웃 구현
2. 메인 대시보드 구현
3. 마이캠퍼스 구현
4. 강의실/주차별 커리큘럼 구현
5. 커뮤니티/설문조사 구현
6. 공통 인터랙션 추가
   - 탭 전환
   - 주간/월간 토글
   - 주차 슬라이더
   - 필터 드롭다운
   - 페이지네이션
7. 반응형/접근성/마감 품질 개선

## 7. 검증 기준

- `DESIGN.md`의 색상/레이아웃/컴포넌트 스타일과 일치하는가
- 주요 페이지가 실제 사이트와 유사한 정보 구조를 갖는가
- 더미 데이터만 바꿔도 화면이 자연스럽게 갱신되는가
- 공통 컴포넌트가 중복 없이 재사용되는가
- 최소한 데스크톱 기준에서 레이아웃이 깨지지 않는가

## 8. 아직 결정해야 할 질문

1. 기술 스택은 무엇으로 할까?
   - 예: React/Vite, Next.js, Vue, 순수 HTML/CSS 등

2. 구현 목표는 어디까지인가?
   - 디자인 목업
   - 프론트엔드 SPA
   - 로그인 포함 풀스택

3. 백엔드는 필요한가?
   - 필요 없음: JSON 더미 데이터 사용
   - 필요함: API 서버와 DB 설계 필요

4. 반응형 범위는 어디까지인가?
   - 데스크톱만
   - 태블릿 포함
   - 모바일까지

5. 실제 edu.ssafy.com과 어느 정도까지 같아야 하나?
   - 분위기와 주요 레이아웃만
   - 페이지별 거의 동일한 UI
   - 기능 흐름까지 유사하게

6. 우선 구현할 페이지는 무엇인가?
   - 메인 대시보드부터
   - 강의실부터
   - 전체 골격부터

## Error Analysis

이전 DESIGN.md 검증에서 발생한 진짜 오류는 `rounded.circle: 50%` 때문이었다.

`@google/design.md` 스펙의 dimension 값은 `px`, `rem`, `em` 단위만 허용한다. `%` 단위는 허용되지 않기 때문에 `50%`가 invalid unit 오류를 만들었다.

해결:

- `rounded.circle: 50%` 대신 `rounded.circle: 999px`로 수정했다.

그 외 WCAG contrast 메시지는 구조 오류가 아니라 경고다. 실제 SSAFY 사이트의 밝은 파란색은 흰색 텍스트와 대비가 낮아서 접근성 기준 AA 4.5:1을 만족하지 못한다. 그래서 원본 색상 `primary`는 유지하되, 컴포넌트에는 접근성용 `primary-accessible`을 추가로 연결했다.

현재 lint 결과는 error 0이며, 남은 warning은 대부분 “정의됐지만 컴포넌트에서 직접 참조되지 않은 색상”이다. 이는 DESIGN.md 설명 본문에서 사용하는 보조 색상까지 토큰화했기 때문에 생긴 경고이며, 구현을 막는 문제는 아니다.
